VBOND
Step 1 (Recon Group)
First find out if your customer's port is already assigned to a bonded group

show dsl-port v$1ST_PORT# basic
show dsl-port v$2ND_PORT# basic
show dsl-bond-interface

Step 2 (Recon)

sh bw-profile

sh svc-tag-action

Step 4 (Create the group)

create dsl-bond-interface v$1ST_PORT#_OF_GROUP admin-state no-alarms subscriber-id /$ISPPACCTID(10 DIGIT ISPP ACCOUNT ID) ds-min-rate 256 us-min-rate 128
set dsl-port v$1ST_PORT# basic dsl-bond-interface v$1ST_PORT#_OF_GROUP
set dsl-port v$2ND_PORT# basic dsl-bond-interface v$1ST_PORT#_OF_GROUP

Step 5 (Cross Connect)

Now Build the Cross Connect:

add eth-svc Data1 to-dsl-bond-interface v$1ST_PORT#_OF_GROUP bw-profile $BW_PROFILE svc-tag-action $VLAN_NAME inner-vlan $C-TAG admin-state enabled
Note/ for NQQ VLAN, use "none" for the inner-vlan

add eth-svc Data1 to-dsl-bond-interface v$1ST_PORT#_OF_GROUP bw-profile $BW_PROFILE svc-tag-action $VLAN_NAME inner-vlan none admin-state enabled
